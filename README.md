📡 Plateforme Web de Supervision Proactive des Abonnements 5G
📌 Contexte & Problématique

Dans un environnement où les infrastructures réseau deviennent de plus en plus complexes et critiques, la gestion efficace du réseau 5G représente un défi majeur pour Tunisie Telecom.

Les solutions actuelles présentent plusieurs limites :

❌ Manque de visibilité globale
❌ Détection tardive des incidents
❌ Outils non centralisés
❌ Difficulté de coordination entre équipes

👉 Il devient donc essentiel de disposer d’une plateforme intelligente, centralisée et proactive pour :

Détecter automatiquement les clients 5G
Contrôler les abonnements par zone
Vérifier la couverture réseau
🎯 Objectif du Projet

Le projet vise à développer une application web permettant :

✔️ Une supervision centralisée du réseau 5G
✔️ Une gestion intelligente des abonnements
✔️ Une amélioration de la qualité de service
✔️ Une détection proactive des anomalies

🧠 Description

La Plateforme web de supervision proactive des abonnements 5G permet :

📍 Détection automatique des clients utilisant la 5G
📊 Visualisation en temps réel des abonnés par zone
📡 Contrôle des abonnements selon la capacité des pylônes
🌍 Vérification de la couverture 5G par zone
⚠️ Détection et gestion des incidents réseau

👉 Cette solution aide à :

Anticiper les problèmes
Optimiser les performances réseau
Améliorer l’expérience client
🏗️ Fonctionnalités Principales
1️⃣ Gestion des utilisateurs 👤

Fonctionnalités :

Création / modification / suppression d’utilisateurs
Authentification (login / logout)
Gestion des rôles (Admin, Technicien, Chef)
Attribution des responsabilités

🎯 Objectif : Sécuriser l’accès et tracer les actions

2️⃣ Gestion des zones réseau 🌍

Fonctionnalités :

Création des zones (Zone A, Data Center, etc.)
Association des équipements aux zones
Consultation de l’état global

🎯 Objectif : Structurer le réseau pour une meilleure supervision

3️⃣ Gestion des données techniques ⚙️

Fonctionnalités :

Stockage des données techniques (IP, débit, latence…)
Association à une zone réseau
Mise à jour automatique ou manuelle

📌 Relations :

Une zone → plusieurs données techniques
Une donnée → une seule zone

🎯 Objectif : Centraliser les informations techniques

4️⃣ Gestion des incidents 🚨

Fonctionnalités :

Déclaration d’incidents
Association à une zone
Définition du niveau de gravité
Suivi de l’état (ouvert, en cours, résolu)
5️⃣ Gestion des interventions 🔧

Fonctionnalités :

Documentation des incidents
Ajout de commentaires et actions
Clôture des incidents

🎯 Objectif : Assurer un suivi efficace et structuré

🤖 Intelligence Artificielle

Le projet intègre un modèle d’IA permettant :

📈 Détection des zones saturées
🔍 Analyse des performances réseau
⚡ Aide à la prise de décision proactive
⚙️ DevOps

Le projet inclut une approche DevOps pour garantir :

🔄 Intégration continue (CI)
🚀 Déploiement continu (CD)
🐳 Conteneurisation avec Docker
📊 Monitoring (ex: Prometheus / Grafana)
🧰 Technologies utilisées (à adapter)
Backend : Spring Boot / Node.js
Frontend : Angular / React
Base de données : MySQL / PostgreSQL
DevOps : Docker, Jenkins
Monitoring : Prometheus, Grafana
IA : Python / Machine Learning
